## 首次使用准备

### 1. 环境配置

确保已按照 README.md 中的说明配置好 `.env` 文件：

```bash
cp .env.example .env
# 编辑 .env 文件，填入 AI API 配置
```

### 2. 启动应用

```bash
npm install
npm run dev
# 访问 http://localhost:3000
```

## 创建和管理项目

### 创建新项目

1. 点击页面右上角的 **"新建项目"** 按钮
2. 填写项目信息：
   - **项目名称**：给你的项目起一个合适的名称
   - **项目需求**：详细描述项目需求，包括：
     - 功能特性
     - 技术要求
     - 界面设计思路
     - 用户体验期望
     - 数据处理需求
   - **提示**：越详细的需求描述会带来越好的结果

3. 点击 **"开始创建"** 按钮

### 项目列表

在首页可以看到所有已创建的项目：

- **项目卡片**：显示项目名称、状态、创建时间
- **状态标识**：
  - 待开始（灰色）：项目尚未开始
  - 运行中（蓝色）：正在进行开发
  - 已完成（绿色）：项目已完成所有步骤
  - 失败（红色）：项目开发过程中出现错误

- **操作**：
  - 点击卡片进入项目详情
  - 点击删除按钮删除项目

## 五步开发流程

### 概述

ZeroAI 采用标准化的五步开发流程，每个步骤都有明确的目标和交付物。

### 步骤 1：需求理解与分析

**目标**：分析用户需求，梳理功能点和技术要求

**操作**：
1. 项目创建后自动进入此步骤
2. 输入区域会显示你在创建项目时填写的需求
3. 点击 **"生成"** 按钮开始需求分析
4. 等待 AI 分析完成

**输出**：
- 需求文档（Markdown 格式）
- 功能特性列表
- 技术架构建议
- 开发计划

### 步骤 2：接口设计

**目标**：设计 API 接口、数据结构和交互模式

**操作**：
1. 步骤 1 审批通过后进入此步骤
2. 点击 **"生成"** 按钮
3. 等待 AI 完成接口设计

**输入**：需求分析结果

**输出**：
- API 接口列表
- 数据结构定义
- 接口文档
- 交互流程图

### 步骤 3：数据库设计

**目标**：设计数据库表结构和关系

**操作**：
1. 步骤 2 审批通过后进入此步骤
2. 点击 **"生成"** 按钮
3. 等待 AI 完成数据库设计

**输入**：需求分析结果

**输出**：
- 数据库表结构
- 字段说明
- 表关系图
- SQL 建表语句

### 步骤 4：业务逻辑设计

**目标**：设计核心业务处理逻辑和流程

**操作**：
1. 步骤 3 审批通过后进入此步骤
2. 点击 **"生成"** 按钮
3. 等待 AI 完成业务逻辑设计

**输入**：接口设计结果

**输出**：
- 业务流程描述
- 状态转换图
- 关键算法说明
- 错误处理机制

### 步骤 5：代码生成

**目标**：根据前面的设计生成完整的源代码

**操作**：
1. 步骤 4 审批通过后进入此步骤
2. 点击 **"生成"** 按钮
3. 等待 AI 完成代码生成

**输入**：需求分析 + 接口设计 + 业务逻辑设计

**输出**：
- 完整的源代码
- 代码文件列表
- 安装和运行说明
- 测试用例

## 步骤操作

### 查看详细信息

1. 点击左侧步骤导航栏查看对应步骤
2. 每个步骤显示：
   - 步骤标题和编号
   - 状态指示器
   - 步骤内容（Markdown 渲染）
   - 操作按钮

### 操作按钮

- **生成**：开始执行当前步骤
- **重新生成**：重新执行当前步骤（覆盖现有结果）
- **审批并继续**：审批当前步骤结果，准备下一步
- **下一步**：（在审批后）进入下一步

### 步骤状态

- **待开始**：灰色，未执行
- **生成中**：蓝色（动画），正在执行
- **待审核**：黄色，等待用户审核
- **已审核**：绿色，已通过审核
- **失败**：红色，执行失败

## 版本管理

### 创建版本

在项目开发过程中的任何阶段都可以创建版本：

1. 点击页面右侧的 **"保存版本"** 按钮
2. 填写版本信息：
   - **版本号**：格式如 v1.0.0
   - **版本名称**：对版本的描述（可选）
3. 点击 **"确定"** 保存

### 查看版本

在版本管理面板可以看到所有已保存的版本：

- 版本号
- 版本名称
- 创建时间
- 是否已发布

### 加载版本

1. 在版本列表中找到要加载的版本
2. 点击版本卡片
3. 点击 **"加载此版本"** 按钮
4. 确认加载操作

### 新版本开发

1. 点击 **"新版本"** 按钮
2. 填写版本信息
3. 系统会重置项目状态，保留需求理解阶段的输入
4. 开始新的开发流程

## 导出文档

### 导出 Word 文档

1. 点击页面右上角的 **"导出 Word"** 按钮
2. 系统会生成完整的项目文档
3. 文档内容包括：
   - 项目信息
   - 每步的标题和内容
   - 所有步骤的 Markdown 内容
   - 版本信息（如已选择版本）

### 文档内容结构

```
[项目名称]
├─ 项目信息
│  ├─ 项目名称
│  ├─ 项目需求
│  └─ 版本信息（如已选择）
├─ 步骤 1：需求理解与分析
│  ├─ 状态：[步骤状态]
│  └─ [步骤内容]
├─ 步骤 2：接口设计
│  ├─ 状态：[步骤状态]
│  └─ [步骤内容]
├─ 步骤 3：数据库设计
│  ├─ 状态：[步骤状态]
│  └─ [步骤内容]
├─ 步骤 4：业务逻辑设计
│  ├─ 状态：[步骤状态]
│  └─ [步骤内容]
└─ 步骤 5：代码生成
   ├─ 状态：[步骤状态]
   └─ [步骤内容]
```

## 高级操作

### 查看详细输入输出

1. 点击步骤内容区域的 **"显示详情"** 链接
2. 在标签页中可以看到：
   - **系统提示**：发送给 AI 的系统提示词
   - **用户输入**：发送给 AI 的用户内容
   - **AI 输出**：AI 的响应结果（默认）
   - **原始响应**：API 接口的原始 JSON 响应

### 查看运行日志

页面底部的日志区域会显示：

- 操作时间和标题
- 详细内容（可展开）
- 日志级别：
  - 🔵 信息：正常操作记录
  - 🟡 警告：需要注意的情况
  - 🔴 错误：错误信息

## 常见问题

### 1. API 调用失败

- 检查 `.env` 文件中的配置是否正确
- 确保网络连接正常
- 检查 API 密钥是否有效
- 查看日志区域的详细错误信息

### 2. 输出结果不符合预期

- 检查需求描述是否足够详细
- 尝试重新生成该步骤
- 修改需求描述后重新开始
- 检查步骤间的关联输入是否正确

### 3. 项目数据丢失

- 项目数据存储在 `zeroai.db` 文件中
- 不要手动删除此文件
- 如果需要备份，复制此文件即可

### 4. 性能优化

- 如果项目较大，AI 响应时间会较长
- 确保网络连接稳定
- 考虑优化需求描述，避免过于复杂的功能

## 最佳实践

### 1. 需求描述技巧

- **详细描述功能**：避免模糊的描述
- **提供示例**：给出具体的使用场景
- **定义边界条件**：说明功能的限制和边界
- **参考类似项目**：如果有类似项目，提供参考

### 2. 开发流程建议

- 每步结果都要仔细审查
- 不要跳过步骤或快速审批
- 遇到不满意的结果及时重新生成
- 合理使用版本管理功能

### 3. 团队协作

- 项目可以通过分享 `.env` 和 `zeroai.db` 进行协作
- 建议定期保存版本快照
- 可以将导出的 Word 文档用于团队审查

### 4. 代码质量

- AI 生成的代码质量取决于需求描述的质量
- 生成后建议进行代码审查
- 根据需要进行手动优化
- 确保遵守团队的代码规范

## 快捷键

- **Ctrl/Cmd + Enter**：快速生成（在输入区域）
- **Esc**：关闭模态框

## 高级配置

### 修改步骤提示词

提示词文件位于 `prompts/` 目录下，每个步骤一个文件。可以根据需要修改：

1. 找到对应的提示词文件
2. 修改内容
3. 保存文件
4. 重新启动开发服务器

### 自定义 AI 模型

修改 `.env` 文件：

```env
OPENAI_BASE_URL=https://your-api.com/v1
OPENAI_MODEL=your-model-name
OPENAI_API_KEY=your-key
```

支持任何兼容 OpenAI 格式的 AI API。
